# Python Module 01 — Обзор и теория (на русском)

Этот модуль — небольшой учебный проект про основы Python и ООП на примере "сада". Все примеры автономные: каждый файл можно запускать отдельно через `python3 имя_файла.py`.

## Что происходит по упражнениям

### ex0: `ft_garden_intro.py`
- Демонстрация переменных и форматированного вывода.
- Функция `ft_garden_intro()` печатает базовую информацию о растении.

### ex1: `ft_garden_data.py`
- Вводится класс `Plant` с полями `name`, `height`, `days`.
- Создаются несколько объектов и выводятся их данные методом `print_info()`.

### ex2: `ft_plant_growth.py`
- Класс `Plant` дополняется методами `grow()` и `age()`.
- Имитируется рост растений за неделю с помощью циклов.

### ex3: `ft_plant_factory.py`
- Появляется счетчик созданных объектов: `Plant._total_plants`.
- Массовое создание растений из списка кортежей.
- Выводится итоговое количество созданных объектов.

### ex4: `ft_garden_security.py`
- Класс `SecurePlant` использует защищенные поля `_height`, `_age`.
- Вводятся сеттеры `set_height()` и `set_age()` с проверкой корректности.
- Демонстрация "защищенного" доступа к данным.

### ex5: `ft_plant_types.py`
- Наследование: базовый `Plant`, производные `Flower`, `Tree`, `Vegetable`.
- Полиморфизм: общий метод `print_info()` работает для разных типов.
- У каждого типа есть свое поведение (цветение, тень, питание).

### ex6: `ft_garden_analytics.py`
- Несколько уровней наследования (`Plant` → `FloweringPlant` → `PrizeFlower`).
- Класс-менеджер `GardenManager` хранит сады пользователей в словаре.
- Вложенный класс `GardenStats` отвечает за аналитику.
- Пример `@classmethod` (фабрика объекта) и `@staticmethod` (утилиты без состояния).

## Ключевые термины и понятия

- Переменная: имя, которое ссылается на значение (`plant_name`, `height`).
- Функция: блок кода, который можно вызвать по имени (`ft_garden_intro()`).
- Класс: шаблон для создания объектов (`class Plant`).
- Объект (экземпляр): конкретный "предмет" созданный из класса (`rose = Plant(...)`).
- Атрибут: данные, принадлежащие объекту (`self.height`).
- Метод: функция внутри класса, работающая с объектом (`grow()`, `print_info()`).
- Инкапсуляция: скрытие деталей реализации, доступ через методы.
  - В Python это соглашение: `_height` означает "внутреннее" поле.
- Наследование: новый класс расширяет базовый (`class Flower(Plant)`).
- Полиморфизм: один интерфейс, разная реализация. В примерах — единый метод
  `print_info()` у разных типов растений.
- `super()`: вызов конструктора (или метода) родительского класса.
- `@classmethod`: метод класса, получает `cls` и может создавать объекты.
- `@staticmethod`: статический метод без доступа к `self` и `cls`.
- `isinstance(obj, Class)`: проверка типа объекта.
- Списки (`list`): упорядоченные коллекции (`plants = [ ... ]`).
- Кортежи (`tuple`): неизменяемые группы значений (`("Rose", 25, 30)`).
- Словари (`dict`): пары ключ-значение (`{"plants": [], "stats": {...}}`).
- Аннотации типов (type hints): подсказки для читаемости и инструментов
  (`def grow(self) -> None`).

## Как запускать

## Замечания по коду (на что обратить внимание)

- В `ex3/ft_plant_factory.py` метод `print_total_plants()` не помечен как
  `@staticmethod` или `@classmethod`. Сейчас он вызывается от класса и работает,
  но вызов от экземпляра (`plant.print_total_plants()`) приведет к ошибке.
- В `ex6/ft_garden_analytics.py` методы `add_plant()` и `help_plants_grow()`
  предполагают, что сад владельца уже создан. Если вызвать без `add_garden()`,
  будет `KeyError`.

В остальном код корректен для учебного демонстрационного уровня.
